## 附：LLM 使用说明

### 任务开始前，必须确认

1. 我当前的任务是否在系统边界之内？
2. 我是否即将触碰任何技术禁区或行为禁区？
3. 有没有不确定的地方需要标记 [PENDING]？
4. 我的实现是否引入了计划外的设计目标？
5. 我是否在"热心过头"——做了没有被明确要求的事？

> 规则：没有被明确要求的设计目标，默认不做。自认为"合理的扩展"不是做它的理由。

### 任务完成后，必须执行

1. 提交 commit，格式：`[模块名] 任务描述`
2. 输出执行摘要（做了什么、为什么这么做、遇到了什么）
3. 更新对应文档
   - 有新接口 → 更新接口文档
   - 有新模块/数据流变化 → 更新架构文档
   - 有关键决策 → 归入 LLM 决策记录
4. 有 [PENDING] 项 → 在状态上下文中标记，等待人确认
